Project Overview

This project was developed as part of a team at The Game Assembly. The game was built around combat, movement and a dark monster-hunting atmosphere. My contribution was primarily centered on player-facing systems and practical features that supported both gameplay and usability.

My Contributions

HUD

Worked on presenting player-facing information such as ammo, health and other UI elements. This involved practical UI implementation and keeping the interface readable during gameplay.

Movement & Controller Support

Worked on movement-related functionality and implemented controller support, including Xbox controller features. This was important for making the game feel responsive and playable.

Health Systems

Worked on health-related systems connected to the player. This included logic that affected gameplay feedback and player state.

Enemy Data Loading

Implemented or worked on loading enemy data from JSON files. This helped support more data-driven workflows and made it easier to work with content in the project.

Challenges & Lessons

One of the main lessons from this project was how much interface and system code can be affected by changes elsewhere in the game. Working on HUD and player-facing features made it clear how important it is to think about dependencies early, especially in a team project where systems evolve over time.
