Spite

A Diablo-style action RPG built in our custom C++ engine (Lamp Engine), where I worked mainly on enemy AI and data-driven enemy behavior.

Role

Gameplay Programmer

Project Type

Team Project

Focus

Enemy AI, Gameplay Systems

Engine

Lamp Engine / C++

Enemy AI & State Machine

My main contribution was enemy AI. I implemented the full logic for the popcorn enemy using a state machine.

The enemy transitions between states like idle, chase, shoot, flee, stunned, and dead based on distance, cooldowns, and player interaction.

I also contributed to behavior on some of the other enemies during development.

Data-Driven Enemies & Tools

Levels were built in Unreal using assets, where we also set values like range, speed, and damage. This data was exported as JSON and loaded into the engine using our level loader.

To support this workflow, I created enemy dummy components so level designers could easily test and iterate on enemy behavior directly in the level.

Engine & Systems

The game was built in our own C++ engine (Lamp Engine).

I worked on systems like an input mapper with JSON-based key bindings and contributed to core engine features such as a state stack and splash screen.

This project was also my first real exposure to working inside and extending a custom engine.

Takeaway

This project gave me my first real experience working with AI systems, animation states, and data-driven gameplay in a custom engine.
