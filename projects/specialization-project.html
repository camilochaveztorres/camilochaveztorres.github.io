City Simulation System

This project is a systemic city simulation focused on autonomous NPC behaviour driven by needs, schedules and world interaction. The goal was to build a believable simulation where characters make decisions, move through the world and interact with dynamic points of interest.

Uses

  • AI decision system (Brain + Actions)
  • Needs system (hunger, energy)
  • Schedule system (day/night behaviour)
  • POI system (homes, work, restaurants, parks)
  • Steering-based movement (arrive + separation)
  • Grid-based world interaction

Overview

The project simulates a small living world where NPCs act autonomously based on internal needs and external conditions. Each character continuously evaluates its hunger, energy and the current time of day to decide what action to perform.

The system is built around a bla bla decision-making, movement and behaviour execution are separated. This allows for clear control flow and makes it easy to expand the simulation with new behaviours and systems.

Systems & Features

AI Decision System

Each NPC uses a Brain system that selects actions based on needs and schedule. Needs (hunger, energy) take priority, while a global clock defines daily routines such as work, leisure and sleep.

Action-Based Behaviour

Behaviour is implemented through modular Action classes such as Eat, Sleep, Work and Relax. Each action handles movement, state transitions and execution over time, creating consistent and extensible behaviour patterns.

Needs System

NPCs have dynamic needs that change over time. Hunger and energy drive decision-making and ensure that characters behave in a believable and reactive way.

POI (Point of Interest) System

The world contains different POIs such as homes, workplaces, restaurants and parks. NPCs interact with these locations to fulfill their needs and perform tasks like working, eating or resting.

Resource-Based Work System

Work locations contain resources that are gradually depleted as NPCs perform tasks. When resources run out, locations are reset or moved, creating a dynamic and evolving simulation.

Movement & Grid System

NPC movement is based on steering behaviours combined with a grid system. The grid ensures stable positioning and interaction with the world, while steering provides smooth and natural movement.
